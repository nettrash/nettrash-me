Support

Applies to: md — the Markdown editor published by nettrash on Google Play (me.nettrash.md).

Need help?

Email nettrash@nettrash.me. Replies usually arrive within a few working days. Please include your Android version (Settings → About phone) and the app version (shown on md's Google Play listing under About this app).

What md is

md is the simplest Markdown editor for Android: write Markdown on one side and watch it render on the other, or flip between Edit and Preview, all in a warm typewriter look. It opens, edits and saves your own files through Android's document system — nothing lives on a server, because there isn't one.

Quick start

  1. Open md. It starts on a fresh, untitled document — just start typing Markdown.
  2. Use the layout switch in the top bar to change how the editor and preview are shown (see below).
  3. Open Save from the overflow (⋮) menu. The first save on a new document asks where to put it and what to call it; after that, Save writes back to the same file.
  4. To edit an existing file, choose Open… from the overflow menu, or tap a .md / .markdown file in a file manager and pick md ("Open with md").

What md does

  • Opens, edits and saves .md / .markdown files (and plain text) through Android's Storage Access Framework — the document is written back to wherever you chose to keep it.
  • Renders a live preview covering headings, bold / italic / inline code / links / strikethrough, bullet / numbered / task lists (with nesting), fenced code blocks, block quotes, GitHub-style tables, and thematic breaks.
  • Offers a width-adaptive layout switch — Edit and Preview on a phone, plus Split (side by side, re-rendering as you type) on a tablet, unfolded foldable, or large screen.
  • Prints the rendered document, or exports it as a themed PDF, through Android's print system; shares the raw Markdown source through the system share sheet.

Common tasks

Create a new document

Choose New from the overflow (⋮) menu. md opens a blank, untitled document. It becomes a file on disk the first time you Save.

Open an existing file

Choose Open… from the overflow menu and pick a file in the system document picker, or open a Markdown / text file from a file manager and choose md. Files opened from a file manager are loaded read-only until you save; use Save As… to write your edits to a new file, or reopen the file through Open… to edit it in place.

Save and Save As

Save writes back to the current file. If there's nowhere writable yet (a brand-new document, or a file opened read-only), it falls through to a Save As… picker so you can choose the name and location. md also flushes a writable, edited document when you send the app to the background, so you rarely lose work.

Switch layout

The segmented control in the top bar switches between Edit, Preview, and — where the screen is wide enough — Split. On a phone in portrait there isn't room for two side-by-side panes, so only Edit and Preview are offered; rotate to landscape, unfold a foldable, or run md on a tablet and Split appears. The choice is remembered.

Print or share

From the overflow menu, Print / Save as PDF… hands the rendered document to Android's print dialog — pick a printer, or choose Save as PDF to export a themed PDF that matches light or dark mode. Share Source… sends the raw Markdown text through the system share sheet.

Frequently asked questions

Where are my files stored?

Wherever you put them. md never moves your documents to a server — there isn't one. Files live in the location you pick in the document picker (on-device storage, Downloads, Google Drive through the picker, an SD card, and so on). See the Privacy Policy for details.

How do I make md the default for a Markdown file?

When you tap a .md / .markdown or plain-text file in a file manager, Android shows an app chooser; pick md (and Always, if your launcher offers it). md opens the file handed to it; use Open… inside md if you'd rather browse.

My Markdown punctuation isn't being "corrected" — is that a bug?

No, that's intentional. md edits Markdown source, so it deliberately turns off autocorrect and smart-punctuation substitution — your ", -- and ... stay literal so the syntax isn't corrupted.

Why doesn't the preview show some advanced Markdown feature?

md's renderer covers everyday Markdown plus the common GitHub extensions (task lists, tables, strikethrough, fenced code). It is a faithful, fast preview rather than a full CommonMark engine, so a few rarely-used constructs may render as plain text. It is the same hand-written parser as the iOS and Mac versions of md, so a document renders the same on every platform. If something common renders wrong, please report it (see Contact).

Does md collect any data about me?

No. md ships with no analytics, no third-party SDKs, and no servers operated by us; it declares no permissions at all and makes no network connections of its own. Full details are in the Privacy Policy.

Troubleshooting

"Save" keeps asking where to save

That happens when the current document has nowhere writable to write back to — a brand-new untitled document, or a file that was opened read-only (for example, handed to md from a file manager via "Open with"). Pick a location once with Save As…; from then on Save writes straight back to that file.

I can't find a file I saved earlier

md writes to the exact location you chose in the picker, not to a fixed "md folder". If you're not sure where it went, use Open… and browse the same provider (Drive, Downloads, etc.) you saved into. md does not keep a recent-files list of its own.

Print / Save as PDF shows nothing to print

The preview is generated from the current document. If the document is empty, the print output is empty too — add some content first. The PDF is produced entirely on-device through an offscreen WebView; no network access is involved.

Privacy & security

md holds no data about you. It declares no Android permissions, requests no INTERNET permission and cannot open a socket, and has no analytics, advertising, or crash-reporting SDKs. Your documents live wherever you saved them through the system picker; md keeps no app-private copy. Full statement: Privacy Policy.

Cross-platform

md on Android is a port of the iOS / iPadOS and macOS app of the same name. The Markdown parser is byte-faithful across all of them, so a document you write on one renders identically on the others. There is no account and nothing to sync — you move documents yourself, as ordinary files, through whatever storage you already use.

Open source

md is built entirely on Google's first-party Jetpack / Compose libraries with a hand-written Markdown parser and no third-party SDKs. Source code and issue tracker live at github.com/nettrash/md.Android for Android; github.com/nettrash/md (iOS / iPadOS) and github.com/nettrash/md.macOS are the Apple versions.

Please file bug reports and feature requests as GitHub issues — they are public by design so other users can see what's already known. For questions about your personal data, email nettrash@nettrash.me directly.
